객체지향프로그램
프로그램을 객체단위로 나눠서 설계하고 구현하는 방식

객체지향 사용하지 않을 때의 문제점
- 데이터와 기능이 따로 놈
- 코드가 많아지면 관리 어려움
- 재사용이 힘듦

객체지향 핵심 개념
1.class
-설계도, 우리가 만드는 참조 자료형
ex ) 이 프로그램에서 사람을 어떻게 정의할건지 
클래스 안에서 정의가능한 내용
- 필드(field) : 각 객체가 가지는 정보(상태,속성)
-> 이름,나이,키,몸무게,성별,주소 등등
-메서드(method): 각 객체가 할 수 있는 동작 기능
-> 먹는다, 공부한다, 운동한다 ....

2.객체(object)
클래스명 변수명 = new 클래스명();
-클래스를 통해서 만들어진 것
-필드에 객체만의 정보를 넣거나 필드의 내용을 사용할 수 있다.
객체변수명.필드명
-클래스에 정의된 기능을 호출하여 사용 가능
객체변수명.메서드명();

3. 객체지향의 4대 특징
- 캡슐화 
데이터를 보호하는 개념
필드 직접 접근하지 못하게 한다.
제공된 메서드를 통해서만 접근할 수 있게 한다.

- 상속
기존 클래스의 내용을 물려받는 기능(필드, 메서드)
코드를 재사용할 때 좋음

- 다형성
하나의 타입으로 여러 객체를 다룰 수 있는 성질
ex 타이어 마다 성능이 다름(비유)

-추상화 
클래스를 단순화 시키는 것
