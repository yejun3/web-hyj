## 프로토콜의 핵심 구성요소

### 구문(Syntax)

- 데이터의 형식이다.
- 즉, **어떻게 생긴 형태로 보낼 것인가**를 정한다

[첫 번째 칸] [두 번째 칸] [세 번째 칸]
- 첫 번째 칸에는 주소
- 두 번째 칸에는 명령어
- 세 번째 칸에는 실제 데이터

[주소]        [명령어]   [데이터]
192.168.0.1   POST       { "name": "홍길동" }
- 192.168.0.1로 보내고
- POST(생성) 요청을 하고
- 홍길동 데이터를 전달한다

프로토콜 의미
- **형태는 있어도 뜻이 없으면 해석이 불가능합니다.**
- 그래서 프로토콜에서는 “이 값은 무엇을 의미한다” “이 값이 오면 이렇게 동작한다” 라는 규칙이 필요


프론트화면에서 벡엔드에게 데이터를 달라고 요청 
백엔드가 DB 데이터 조회 요청 데이터 있으면 응답
백엔드가 프론트화면에 응답하여 데이터 화면에 구현

**요청메서드**
- 클라이언트가 서버에 요청할 때 다음과 같은 메서드를 씁니다.
    - GET
    - POST
    - PUT
    - DELETE
- 이 단어들은 단순한 문자열이 아닌, 각각 의미가 정해져 있습니다.
**GET**
- 데이터를 조회하라는 의미입니다.
**POST**
- 새로운 데이터를 생성하라는 의미입니다.
**PUT**
- 기존 데이터를 수정하라는 의미입니다.
**DELETE**
- 데이터를 삭제하라는 의미입니다.

타이밍(timing)
데이터를 언제 보내고, 어떤 순서로 보내고, 얼마나 빠르게 보내며, 
얼마나 기다릴지 정하는 규칙

필요 이유
나올 수 있었던 오류상황
- 한쪽이 계속 데이터를 보냄
- 상대는 아직 준비가 안됨
- 순서가 뒤죽박죽
- 응답이 안 오는데 계속 기다림

이에따라 규칙 필요
#### 타이밍이 다루는 핵심 4가지

언제 데이터를 보낼 것인가
얼마나 기다릴 것인가
얼마나 빠르게 보내는가
어떤 순서로 보내는가

프로토콜 동작 예시

 DNS
- 도메인 이름을 IP 주소로 바꾼다.
- `www.example.com` → `93.184.216.34`

 HTTP 또는 HTTPS
- 브라우저가 서버에게 웹페이지를 요청한다.
- 브라우저: 메인 페이지 주세요
- 서버: HTML 문서 보내드리겠습니다

f12 :  개발자도구 여는 키
cURL은 서버에 요청을 보내고 응답을 받아오는 명령어 도구

터미널에서 웹 요청을 보내는 프로그램

IP(Internet Protocol)
주소 -데이터를 목적지까지 보내는 역할

IPv4
- IPv4라는 가장 널리 쓰이는 IP 주소체계
-4마디 총 32비트 2의 32 제곱 43억개 고유주소 생성 가능

IPv6
- IPv4와 달리 콜론 `:` 을 이용한 구분을 한다.
- 16진수로 표현을 한다.
- 128비트의 공간을 가지고 있기 때문에 생성 가능한 주소가 2의 128제곱으로 무제한에 가까운 숫자만큼 할당이 가능

 TCP (Transmission Control Protocol)
- TCP는 정확하고 안정적으로 데이터를 보내는 프로토콜

UDP (User Datagram Protocol)
- 빠르게 보내는 데 집중하는 프로토콜

HTTP (HyperText Transfer Protocol)
- 웹 브라우저와 웹 서버가 대화하는 규칙**이다.
역할
- 브라우저가 서버에 요청한다
- 서버가 응답을 준다
