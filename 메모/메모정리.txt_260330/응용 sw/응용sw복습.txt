OSI 7계층
-데이터다 포장-> 전달 -> 해석되는 과정

1. 물리계층
데이터를 0과 1의 전기신호로 변환하여 전달
프로토콜은 없음
데이터 상태: 비트(bit)
장비
- 허브(Hub): 여기에 연결된 장치에 데이터를 전달
- 리피터(Reapeter): 신호를 증폭시키는 역할
- 케이블(Cable): 데이터 이동 통로

2.데이터 링크 계층
같은 네트워크 내부로 전달
MAC 주소 기반으로 정확하게 전달
ㄴ네트워크에 연결할 수 있는 기기에 할당된 주소
데이터 상태: (프레임)
장비
- 스위치(Swich) 
- 브릿지

3.네트워크 계층
-목적지까지 가장 빠르게 데이터를 보내는 계층
-최적의 경로를 설정해야함
프로토콜
- IP : 주소지정+ 데이터 전달
- ICMP : 네트워크 상태 확인(ping 명령)
- ARP : ip -> MAC주소로 변환
데이터 상태: 패킷
장비
- 라우터(Router) : 네트워크간 이동
ㄴ 하나의 ip를 나눠갖는다. 예시) 101동 101호, 101동 102호.. 
// 허브나 스위치는 한 공간에서만 통신
// 요즘은 공유기가 라우터의 기능을 포함하고 있음

 -서브넷마스크
 - IP 주소에서 어디까지가 네트워크이고 어디까지가 컴퓨터인지 구분하는 값
    255.255.255.0 -> 11111111.11111111.11111111.00000000
       1-> 네트워크 부분
        0-> 호스트 부분

    예시)
     ip주소 :192.168.1.10
     서브넷 마스크 :255.255.255.0

      네트워크 부분 : 192.168.1
      호스트 : 10

 - 게이트 웨이(통로)
   같은 네트워크 밖으로 나갈떄 거치는 출구
    내  PC -> 게이트웨이 -> 인터넷

4.전송계층
- 데이터를 안전하게 보내는 핵심 계층
  포트번호를 사용해서 통신

 프로토콜(어떤 프로토콜을 선택할지는 개발자가 소프트웨어를 만들 떄 선택)
 - TCP : 데이터 확인, 순서보장(웹, 로그인, 파일 다운로드)
 - UDP : 빠름,순서보장 안함, 유실가능(게임,스트리밍,음성통화)

데이터 상태 : 세그먼트(Segment)
-데이터를 나눠서 보내기 위해 만든 작은 데이터 조각
- TCP/UDP 방식으로 데이터를 쪼개고 헤더를 붙임

사용자가 데이터를 보냄
TCP가 데이터를 적당한 크기로 나눔 
TCP헤더
ㄴ 출발지 포트 : 보내는 프로그램 번호
ㄴ도착지 포트 : 받는 프로그램 번호
ㄴ 순서번호 : 데이터 순서 확ㅇ니
ㄴ 확인응답번호 : 어디까지 받았는지 확인
ㄴ 제어 플래그 : 연결상테 확인 
ㄴ윈도우 크기 : 한 번에 받을 수 있는 양
ㄴ체크섬: 데이터 오류 확인

5.세션계층
- 컴퓨터끼리 연결을 만들고 유지하거나 끊는 역할

세션설정
통신을 시작하기 전에 두 시스템 사이에 대화할 수 있는 상태를 만든다.

세션유지
통신 중간에 연결 상태를 관리한다.
로그인 - 장바구니 상품담기 - 다른페이지로 이동해도 로그인 상태가 유지, 장바구니 유지됨
 - 세션유지방법
  1. 쿠키발급 -> 사용자가 가지고 있기 때문에 보안 취약
  2. 세션객체 -> 개인이 가지고 있는 것보다는 보안이 좀 나음( 서버에 부담이 감)
  3. 토큰 발급-> 암호화를 해서 발급

세션종료
통신이 끝나면 세션을 정리한다.
로그아웃 - 브라우저 닫기 - 일정시간 끝나면 세션 종료

6.표현계층
-데이터의 암호화/압축/번역을 하는 계층
프로토콜
SSL/TLS : 데이터를 암호화해서 도청 방지 (HTTPS의 핵심)
JPEG/MP3: 데이터 형식을 정의

7.응용계층
-사용자가 직접 사용하는 웹, 앱 같은 기능
프로토콜
HTTP: 웹 사이트를 접속할 떄 사용하는 규칙
"이 페이지를 주세요(요청)" -> "여기 있습니다.(응답)"
FIP: 파일 업로드/다운로드
DNS : 도메인 -> IP주소 변환

미들웨어
- 서로 다른 프로그램이나 연결을 도와주는 중간 소프트 웨어

웹 서버 미들웨어
- NginX
-  Apache HTTP Server
- 사용자 요청이 들어오면   어떤 서버로 보낼지 안내 해주는 역할

WAS(Web Application Server)
- java 웹 어플리케이션 실행
- 세션관리
- DB 연동지원

DB 미들웨어
- 어플리케이션과 DB연결
- SQL 실행지원
- DB 연결 재사용
- 객체와 테이블 매핑

메세지 지향 미들웨어(Message Orinted Middleware)
* 메세지 : 전달할 내용( 시스템과 시스템 사이 주고받는 데이터 묶음)
- 보내는 사람은 우체통의 편지를 넣고, 받는 사람은 나중에 꺼내서 처리한다.

TP Moniter
- 대량 트랙잭션 처리
- 트랜잭션 안정성 보장
- 동시 사용자 처리
- 금융, 공공 시스템에서 많이 사용

EAI
기업 내부의 관련없는 여러 시스템을 통합하는 기술
인사 시스템, 회계시스템, 물류시스템, 고객관리 시스템과 같은 시스템들이 데이터를 주고받도록 연결해준다.

ESB
여러 시스템을 버스처럼 연결하는 구조 
버스터미널처럼 타고 싶은 사람만 데이터를 취하는 구조

데이터 : 그냥 객관적인 값
DB : 데이터들의 모음
DBMS : DB를 관리하는 프로그램
ㄴ계층형 : 구조가 단순하고 조회가 빠르다. 구조변경이 어렵다. (윈도우 형식) -> 구조가 단순하고 계층이 명확할 때 사용
ㄴ네트워크 : 데이터를 그래프로 저장한다. N:M 관계가 가능하다. -> 관계가 복잡하고 서로 많이 연결될 떄
ㄴ관계형 : 일반적인 서비스 대부분
ㄴ객체형 : 객체지향 구조를 그대로 쓰고싶을 때

DDL : 데이터 베이스 객체를 조작하는 SQL
ㄴcreate ; 생성
ㄴ drop : 삭제
ㄴ alter : 수정
DML : 데이터를 조작하는 SQL
ㄴ select : 조회
ㄴ insert : 추가
ㄴ update : 수정
ㄴ delete : 삭제
DCL : 권한을 조작하는 SQL
ㄴ grant
ㄴ revolk
TCL : 트랙잭션을 제어하는 SQL
ㄴ commit
ㄴ rollback

모델링
- 테이블을 만들기 전에 요구사항을 잘 분석하고 무결성을 지키기 위한 계획

요구사항 분석 
개념모델링 - 엔티티 구성 , ERD를 그린다.
논리모델링 - 정규화, 속성 자료형, 제약조건 설정
물리모델링 - 실제로 테이블을 만드는 과정

정규화
- 이상현상을 제거하기 위해 테이블을 분리하는 것
ㄴ 삽입이상 : 데이터를 추가할 떄 필요없는 내용까지 추가되는 것
ㄴ 갱신이상 : 중복되는 데이터가 있을 떄 일부분만 수정하는 것
ㄴ 삭제이상 : 데이터를 삭제할 때 필요한 내용까지 삭제되는 것
