데이터 모델링
- 현실 세계의 업무내용을 데이터베이스에 저장할 수 있도록 데이터 구조로 설계하는 과정

모델링이 필요한 이유 
이상현상의 발생을 막아야한다.
1.데이터 중복이 발생할 수 있다.
2.갱신 이상 -> 같은 정보가 여러 개 저장되어 있을 때 전부 수정하지 않아서 데이터의 불일치가 일어나는 현상
3.삭제 이상 -> 필요없는 데이터를 지우다가 필요한 데이터까지 삭제하는 경우
4.삽입 이상 -> 필요한 데이터를 추가하려다 필요없는 데이터까지 추가하는 경우
//모델링을 하지 않을 경우 발생될 수 있는 문제

데이터 모델링 과정
1.요구사항 분석
-업무에서 어떤 데이터를 관리해야 하는지 파악하는 단계
-주요 명사를 찾아서 -> 엔티티로 만든다.

2.개념적 모델링 
-업무내용을 큰 그림으로 표현하는 단계이다.
-주로 ERD를 사용한다.

주요 구성요소 
엔티티(Entity) : 관리해야 할 대상
속성(atrtribute) : 엔티티가 가진 정보
관계(Relactionship) : 엔티티 간 연결
식별자(Idrntifier) : 데이터를 구분하는 값

엔티티 간 관계설정
-관계를 문장으로 처리
학생은 수강신청을 한다.
강사는 수강신청을 받는다.
강사는 수업을 담당한다.

관계 차수 정하기
1:1, 1:N, M:N 중 어떤 관계인지 정한다.

학생 1명은 여러 수강 신청을 할 수 있다. -> 학생1 : 수강신청 N
수업 1개는 여러 수강신청을 받을 수 있다. -> 수업1 : 수강신청 N
강사 1명은 여러 수업을 담당할 수 있다. -> 강사1 : 수업 N

학생 N : 수업 M 
실제 설계에서는 중간 엔티티인 수강신청으로 풀어준다.
학생1: N 수강신청 N :1 수업

논리 모델링 
-개념적 모델을 실제 테이블 구조에 가깝게 바꾸는 단계
-ERD를 가지고 테이블을 설계
-테이블의 이름이나 속성의 이름을 정하는 단계
-제약조건(PK,FK 등..), 정규화

정규화 (NF)
실무에서는 보통 3차까지 다룸 -> 더 하면 오히려 조회속도가 느려짐

1NF -> 속성 하나에는 하나의 값만 들어가야한다.
학생번호     이름               전화번호
1               김민수           010-1111, 010 -7777 //1차 정규화 위반
2               이영희            010-4545

-> 학생번호    이름               전화번호
1                김민수             010-1111
1                김민수             010-7777
2                이영희             010-4545

제 2정규형(2NF) -> 복합키에서 일부 컬럼에만 의존하는 컬럼을 제거 (부분함수 제거)

학생번호  과목번호   이름         과목명      성적
1               101       김민수       java          A
1               102       김민수       SQL          B

학생번호 + 과목번호 -> 성적
학생번호 -> 이름
과목번호 -> 과목명

관련 있는 것들끼리 쪼갠다.
학생            
학생번호  이름
1            김민수

과목
과목번호        과목명
101               java
102               SQL 

수강내역
학생번호   과목번호   성적
1               101          A
1               102          B

제 3규정형(3NF) -> 일반 컬럼이 다른 일반 컬럼을 결정하면 안된다.
(이행 함수 종속 X->Y, Y->Z, X->Z) // 이행 함수 종속이 일어나서는 안됨
학생번호           이름              전공번호         전공이름
1                     김민수               10                 컴공
2                     이영희               20                디자인

중간에 다른 컬럼을 거쳐서 결정되면 분리

전공

  10                 컴공

BCNF(Boyce-Codd Nomal Form)
- 제 3정규형을 좀 더 엄격하게 적용한 정규형
- 모든 함수적 종속에서 반드시 후보키여야 한다.
*결정자 : 관계형 데이터베이스에서 다른 속성의 값을 유일하게 결정짓는 속성들의 집합을 의미한다.(A ->B)
*후보키 : PK가 될 수 있는 속성

제 3정규형을 지켜도 이상 현상이 남아있을 수 있다.
학생명      과목          강사

김민수      OS            박강사
이영희      DB            이강사

subject가 결정자인데 후보키가 아님 

과목 
과목명      강사
DB           이강사
OS           박강사

김민수      DB

SQL 연산자
1. BETWEEN : A와 B사이의 값을 조회할 때 사용
- 같은 속성 사이의 값만 사용가능
2. IN : OR을 대신해서 사용하는 연산자

LIKE
-문자열 안에서 특정 글자나 패턴을 찾을 때 사용하는 조건문
SELECT 컬럼명
FROM 테이블명
WHERE 컬럼명 LIKE '패턴'

SELECT * FROM FILM WHERE TITLE = 'LOVE'
- 이 SQL은 제목이 정확히 LOVE인 영화만 찾는다.

LIKE는 포함 여부, 시작문자 ,끝문자 ,특정글자 패턴을 찾을 수 있다.

% : 글자 수와 상관없이 아무 문자나 여러 개 올 수 있다는 뜻

'A%' : A로 시작하는 데이터 
'%A' : A로 끝나는 데이터
'%A%' : A를 포함하는 데이터
'A%B' : A로 시작하고 B로 끝나는 값

_ : 문자 한개를 의미함
'A_' : A로 시작하고 전체 길이가 2인 값
'A__' : A로 시작하고 전체길이가 3인 값
'__A' : A로 끝나는 전체길이가 3인 값
'_A_' : 전체길이가 3이고 A가 중간에 있는 값
