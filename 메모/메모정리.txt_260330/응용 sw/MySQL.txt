MySQL 
관계형데이터베이스(RDBMS)
- 테이블 형태로 데이터를 관리
-행(row) : 가로 데이터셋(한 건의 데이터) 튜플,레코드 라고도 함
ㄴ 행의갯수 : 카디널리티
-열(column): 세로 데이터셋(한 가지의 데이터) 속성(attribute)

SQL이란?
-데이터베이스에 저장된 데이터를 다루기 위한 언어(추가,조회,수정,삭제)

SQL의 종류

DDL : 데이터베이스 객체를 조작하는 SQL
- create (데이터배이스 객체 생성), 
- alter(데이터 수정)
- drop(데이터 삭제)

DML : 테이블 내의 데이터를 조작하는  SQL
- select : 테이블 내의 데이터를 조회하여 보여준다.
- insert : 테이블에 데이터를 추가한다.
- update : 테이블 내 데이터를 수정한다.
- dlelete : 테이블 내에 데이터를 삭제한다.

DCL : 권한권리 명령어
- grant : 권한을 주는 명령어
- revoke : 권한을 회수하는 명령어

TCL : 트랜잭션 명령어
- commit
- rollback

트랜잭션 : 여러 SQL 작업을 하나의 작업단위로 묶는 것
EX) 계좌이체
A계좌에서 10만원 이체
B계좌에서 10만원 입금

MySQL은 기본적으로 AUTO COMMIT 상태로 되어있다.

제약조건
- 테이블에 잘못된 데이터가 들어가지 않도록 막는 규칙
pk - primary key (기본키), null불가 , 중복불가
NOT NULL - 값을 비워두지 마라
UNIQUE - 중복된 값을 넣지 마라
FK - foreign key (외래키) 다른 테이블의 기본키를 참조하는 제약조건
ㄴ 없는 데이터를 참조하지 못하게 하는 규칙

check - 특정 값만 조건을 만족하도록 제한하는 제약조건
default - 값을 입력하지 않았을 떄 자동으로 들어갈 기본값 설정

모델링 
- 현실세계의 데이터를 데이터베이스 구조로 설계하는 과정

요구사항 분석
개념모델링 - 엔티티의 결정(ERD 작성)
논리모델링 - ERD를 자료형이나 제약조건 설정, 정규화 (1~5차)
물리모델링 - 살제 테이블로 만드는 과정

SQL 
단일 행 함수 - 1개의 행 값이 함수에 적용되어 1개의 행을 반환하는것
집계함수 - 1개 이상의 행 값이 함수에 적용되어 1개의 행을 반환한다.

문자함수
LENGTH() : 바이트 수 반환
CHAR_LENGTH() : 글자 수 반환
UPPER(), LOWER : 대문자로 변환, 소문자로 변환
TRIM(),LTRIM(),RTRIM(): 양쪽 공백제거, 왼쪽공백제거, 오른쪽공백제거
LPAD(),RPAD(): 문자열의 자리수를 지정, 남으면 특정문자열로 채운다.
SUBSTRING(), SUBSTR(), MID() : 문자열 자르기
* MySQL에서 인덱스는 1부터 시작한다.

LEFT(),RIGHT() : 문자열을 왼쪽/오른쪽부터 내가 원하는 만큼 자른다.
INSTR() : 문자열의 위치번호 찾기
CONCAT() : 나열된 문자열의 연결 //null을 포함하고 있으면 null이 나옴
CONCAT_WS() : 여러값을 하나로 문자로 합칠 떄 구분자를 자동으로 넣어주는 함수
REPLACE() : 문자열에서 특정 문자열을 다른 문자열로 교체(등장하는 모든 문자열을 교체해준다.)
ㄴ 없으면 원래 문자열 그대로 반환한다.
SUBSTRING_INDEX(문자열, 구분자, 개수) : 문자열을 특정 구분자를 기준으로  잘라서 원하는 부분만 가져오는 함수
