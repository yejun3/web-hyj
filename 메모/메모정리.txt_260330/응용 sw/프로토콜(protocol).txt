프로토콜(protocol)
- 컴퓨터끼리 대화(통신)할때 지켜야 하는 규칙

핵심 구성요소
- 구문(데이터의 형식)
- 의미(보내진 데이터가 무엇을 의미하는지, 어떤 동작을 해야하는지)
ㄴ요청 메서드 GET, POST, PUT, DELETE
ㄴ응답 코드 2XX, 3XX, 4XX, 5XX
- 타이밍(언제보내고, 순서, 속도)

대표적인 프로토콜의 종류
HTTP, HTTPS -> 브라우저가 서버에 페이지를 요청할 때
FTP -> 파일 전송 (개발 : 윈도우, 운영 : 리눅스)
TCP -> 데이터를 정확하고 안정적으로 보낼때 
UDP -> 빠르게 보내는데 집중하는 프로토콜

OSI 7계층
- 컴퓨터끼리 통신할 때 7단계로 나눠서 처리하는 구조
1계층 : 물리계층
- 데이터를 실제로 보내는 계층
- 데이터의 상태가 bit(0과 1로 이루어짐)이다.
- 전기, 빛, 전파와 같은 순수 신호 상태
- 안에 뭐가 들어있는지는 모름
장비
- 허브 : 여러 컴퓨터를 연결하는 장치
- 리피터 : 약해진 네트워크 신호를 다시 강하게 만들어주는 장치

2계층 : 데이터 링크 계층
- 같은 네트워크 안에서 데이터가 제대로 가도록 처리하는 계층
- 데이터 형태 : Frame
ㄴ Packet에 MAC 주소와 오류 검출 정보가 붙은것
- MAC 주소 : 인터넷을 연결할 수 있는 기기에 할당되어있는 고유한 ID
장비
- 스위치 : 필요한곳에만 데이터를 전송하는 장비
ㄴ 스위치가 어느 장비로 데이터를 전송할지 판단할 때 사용하는 것이 MAC 주소이다.
- 브리지

3계층 : 네트워크 계층(Network Layer)
- 서로 다른 네트워크를 넘어 목적지까지 가는 길을 찾는 계층
- ip 주소를 사용한다.
데이터 상태 : Packet
ㄴ Segment에 출발지와 목적지의 IP주소 정보가 붙은 상태
장비
라우터 : 서로 다른 네트워크를 연결해주는 장비
L3 스위치 : 스위치의 기능에 라우터 기능을 합친 장비

4계층 전송계층(Transport Layer)
- 데이터를 정확하고 안정적으로 전달할지, 빠르게 전달할지 결정하는 계층
데이터 상태 : Segment(상위계층에서 내려온 Data에 프로세스 간 통신을 위한 정보가 붙은 상태)
장비
L4 스위치 : IP + 포트번호를 기준으로 트래픽을 분해하는 장비
로드 밸런서 : 서버 여러 대에 트래픽을 균등하게 나눠주는 장비
일부 방화벽

5계층 : 세션 계층(Session)
- 통신의 시작, 유지, 종료를 관리하는 계층
데이터의 상태 : Data

6계층 : 표현 계층(Presentation)
- 데이터의 표현 방식을 맞춰주는 계층
- 데이터를 보기 좋게 변환한다.
데이터 상태
Data

7계층 : 응용 계층(Application)
- 사용자가 직접 만나는 계층
- 웹 브라우저, 메일 프로그램, 파일 전송 프로그램 등이 관련이 있다.
데이터 상태
Data : 여기서는 아직 사람이 이해할 수 있는 실제 내용
하드웨어 장비보다는 소프트웨어 서비스와 더 관련이 깊다.

백그라운드로 돌리기

데이터 전송 흐름
7->6->5->4->3->2->1
1. 응용 -> HTTP 요청을 생성
2. 표현 -> 암호화/변환
3. 세션 -> 연결 생성
4. 전송 -> TCP + 포트
5. 네트워크 -> IP주소
6. 데이터링크 -> MAC주소
7. 물리 -> 신호로 전송

데이터 받을 때
1-> 2-> 3-> 4-> 5-> 6-> 7
1. 신호 수신
2. 프레임 확인
3. IP확인
4. 포트 확인
5. 세션 확인
6. 데이터 해석
7. 프로그램 전달

통신은 요청 <-> 응답
요청(request)을 하는쪽을 클라이언트(Client)라고 한다.

응답(response)을 하는쪽을 서버(Server)라고 한다.

http는 한 번 연결을 하고 나면 기억을 하지 않는다.
1. 클라이언트가 요청을 보냄
2. 서버가 응답을 보냄
3. 연결 종료(끝)

서버 입장에서는 "이 사용자가 방금 전에 뭘 했는지를 모른다."
로그인 요청 -> 성공
다음 페이지를 달라고 요청하는 데 서버는 "너 누군데?"(다시 인증을 요구)

로그인이 유지가 안됨
장바구니가 유지가 안됨
사용자 구분 불가능

상태를 기억하기 위해 별도의 기술을 활용해야 한다.

1. 쿠키
브라우저에 저장되는 데이터
서버 -> 쿠키를 발급
브라우저는 -> 요청할 때마다 쿠키를 같이 자동으로 전송

2. 세션(session)
서버가 사용자 상태를 기억
java에 Session클래스가 있다.
세션 객체를 만들어서 사용자 상태를 저장한다.

3. 토큰(jwt)
- 요즘 가장 많이 사용하는 방식
로그인 성공 -> 토큰 발급
클라이언트가 저장
요청시 토큰을 같이 보냄

MSA(Microservice Archietcture)
- 분산시스템을 구성하기 위해 하나의 큰 어플리케이션을 여러 개의 작은 서비스로 나눠서 개발하고 운영하는 구조
- 각각의 서비스는 독립적으로 실행된다.
- 서비스 끼리는 네트워크로 통신(HTTP 등)
- 배포도 각각 따로 가능함

분산 시스템 SW

시스템
여러 요소가 모여 하나의 기능을 수행하는 구조

분산
한 곳에 몰아두지 않고 여러 곳에 나누는 것

SW
그 구조가 실제로 동작하도록 만드는 소프트웨어

여러 컴퓨터에 기능을 나누어 배치하고, 이들이 서로 통신하면서 하나의 시스템 처럼 동작하게 하는 소프트웨어

왜 분산 시스템 SW가 필요할까
- 한 대의 서버로도 프로그램은 만들 수 있다.
사용자가 많아지고, 기능이 많아지고, 장애에 강해야 하면 한 대만으로는 한계가 생긴다

쇼핑몰에 사용자가 갑자기 10만명이 몰리면, 한 서버가 감당하기 어렵다.

서버1 : 로그인
서버2 : 상품 목록
서버3 : 결제
서버4 : 리뷰

한 대의 서버가 있으면 그 서버가 죽는 순간 서비스 전체가 멈춘다
여러 서버로 나누면 일부 서버가 고장나도 서비스가 완전히 멈추지 않도록 설계할 수 있다.

분산 시스템 SW 구성 요소

1. 어플리케이션 서버
- 실제 비즈니스 로직을 처리하는 서버
EX) 스프링부트, Django

※ 비즈니스 로직 : 백엔드 기능(규칙/판단/계산)

2. 데이터베이스 서버
- 데이터를 저장하는 서버
MYSQL
ORACLE

분산환경에서는 데이터베이스도 하나만 쓰지 않고 나누거나 복제하기도 한다.

3. 로드밸런서
들어오는 요청을 여러 서버에 나누어주는 장치 또는 소프트웨어
NginX

단점
- 구조가 복잡해진다.
- 네트워크를 타다보니, 지연, 끊김, 타임아웃 문제가 생길 수 있다.
- 데이터베이스를 나누다보면 데이터가 안맞을 수 있다.

EAI /ESB
서로 다른 성격의 시스템을 연결해주는 방식

회사에서는 프로그램을 하나만 쓰지 않는다.
홈페이지는 주문을 받고
창고 시스템은 재고를 관리하고
회계 시스템은 매출을 계산하고
배송 시스템은 배송 상태를 관리한다.

이 시스템들이 서로 연결되지 않으면 사람이 직접 옮겨 적거나, 파일을 수작업으로 옮겨야 한다.

EAI (Enterprise Application Integration)
회사 안의 여러 프로그램을 서로 연결해서 하나처럼 동작하게 만드는 방식

회사 중앙에 연결 담당 센터를 하나 두고 모든 부서가 그 센터를 통해 소통하는 구조

구조
허브 앤 스포크(Hub and Spoke)
Hub는 중심
Spoke : 자건거 바퀴 살처럼 바깥으로 연결된 구조

ESB (Enterprise Service Bus)
EAI가 중앙 허브 중심이라면 ESB는 공통 통신 통로(버스)를 만들어서 각 시스템이 버스를 이용해 유연하게 통신하는 방식

특징
표준 기반 통신
서로 다른 시스템들이 같은 방식으로 통신할 수 있게 한다.

느슨한 결합
시스템끼리 아주 강하게 묶이지 않게 만든다.

메시지 중심 구조
데이터를 메시지 형태로 주고받는다.

라우팅 기능
어떤 메시지를 어떤 시스템으로 보낼지 결정

변환 기능
데이터 형식을 바꿔준다.

확장성이 좋다.
서비스가 늘어나도 비교적 유연하게 붙일 수 있다.

핵심 역할
1. 자원 할당
- 어떤 작업에 CPU, 메모리를 얼마나 줄 지 결정

2. 자원 분배

3. 모니터링
CPU, 메모리, 네트워크 상태 확인

4. 부하 분산

5. 자동 확장
트래픽이 많으면 서버 추가
트래픽이 적으면 서버를 줄인다.

- 전세계 여러곳에 서버를 두고 사용자에게 가장 가까운 서버에서 콘텐츠를 제공하는 시스템

사용자 -> CDN -> 원본 서버

필요 시 원본 서버 요청

데이터베이스 기초 활용하기
데이터(Data)의 개념
- 객관적인 사실이나 값을 나타내는 것으로 숫자, 문자, 이미지, 비디오 등과 같은 형태로 나타낸다.
- 이러한 데이터들은 특정 의미를 부여받아 정보가 되기도 하고 더 나아가 지식으로 발전할수도 있다.

데이터베이스
파일을 저장하고 관리하는 특수한 형태로 데이터를 공용으로 활용하기 위해 통합하여 저장한 운영 데이터의 집합

데이터베이스의 특징
실시간 접근성
- 요청받은 데이터는 실시간으로 처리되고 결과를 반환해야 한다.
계속적 변화
- 저장된 데이터는 입력, 수정, 삭제에 의해 지속적으로 변화한다.
동시 공용
- 서로 다른 목적의 SW 및 사용자에 의해 동시 공용 가능
내용에 의한 참조
- 데이터 참조는 데이터의 주소가 아닌 저장된 값에 의해 처리된다.

데이터베이스 관리 시스템(DBMS: DataBase Management System)
- 다수의 응용 소프트웨어 및 사용자가 데이터베이스에 접근하여 저장되어 있는 데이터베이스를 필요야 따라 사용할 수 있도록 관리해주는 시스템

데이터베이스 관리 시스템의 기능
동시성 제어
- 다수 트랜잭션의 동시 처리로부터 데이터 무결성 확보를 위한 제어 수행
ㄴ 여러 사용자가 같은 데이터에 동시에 접근할 때
A가 같은 계좌에서 5만원 출금
B가 같은 계좌에서 3만원 출금
회복관리
- 시스템 오류 및 장애로 인한 데이터 손실 및 결함의 대응
성능
- 데이터 처리 속도 확보를 위한 실행 계획의 최적화
보안 관리
- 비인가 사용의 접근 제어 및 중요한 정보의 암호화

데이터베이스 관리 시스템의 유형
계층형
- 트리구조로 데이터 저장(부모-자식 관계가 형성)
- 자식은 하나의 부모만 가진다.
- 데이터 접근속도 빠름, 구조는 단순함
- 구조 변경이 어려움, 유연성 부족, 관계가 복잡해지면 관리 어려움

네트워크형
- 그래프 구조, 데이터간 다대다 관계가능
- 데이터 노드와 관계를 링크로 연결
- 복잡한 관계표현 가능, 데이터 무결성 높음
- 구조가 복잡함, 설계, 유지보수가 어려움

관계형
- 테이블 형태로 데이터를 저장
- 행과 열로 구성이 되어 있다.
- SQL 사용, 스키마 기반, 정규화 지원
- 직관적, 표준 SQL지원, 데이터 무결성 보안 우수
- 대규모 데이터 처리시 성능 저하 가능

※스키마
- 데이터를 어떻게 저장할지 미리 정의해둔 구조

종류
Oracle
MySQL
PostgreSQL
MSSQL

객체지향형
- 객체 단위로 데이터 저장,  속성과 메서드 포함
- 객체지향 언어와 연동 용이, 상속 다형성 지원
- 복잡한 데이터 구조 직접 표현 가능, 재사용성 높음
- 표준화 부족, 학습 곡선이 높다.

객체관계형
- 관계형 데이터베이스에 객체지향 개념을 추가
- 대용량 멀티미디어 처리 가능
- 기존 RDBMS 장점을 유지, 복잡한 데이터 처리 가능
- 구현이 복잡, 성능 최적화 어려움

DBMS별 특징
관계형
MySQL :  오픈소스, 널리 사용됨, 안정성 높음
MariaDB : MySQL 포크 버전, 오픈소스, 성능 개선
PostgreSQL : 고급 SQL 기능, ACID 준수, JSON 지원, 확장성 높음

ACID
A(Atomicity 원자성) -> 트랜잭션이 여러개일 때 전부 성공하거나, 전부 실패해야 한다.

C(Consistency 일관성) -> 트랜잭션 전후에 데이터 규칙이 유지되어야 한다.

ex) 계좌 잔액은 0원 이상이어야 한다.

I(Isolation 격리성) -> 여러 트랜잭션이 서로 간섭하지 않아야 한다.

ex) A가 데이터를 수정중일 때 B가 그 데이터의 상태를 중간에 확인할 수 없다.

D(Durability 지속성) -> 성공한 작업은 절대 사라지면 안된다.

ex)결제기록이 사라지면 안됨

Oracle : 대규모 기업 환경에 강함, 보안,백업, 복구 기능 강력함
SQLite : 서버 없이 사용 가능, 경량, 임베디드DB

NoSQL
Redis : 초고속 처리, 캐싱,세션 관리에 최적화

트랜잭션
데이터베이스 관리시스템에서 저장된 데이터의 상태를 바꾸기 위해 수행하는 작업의 논리적 단위

논리적 단위의 의미는 여러 개의 작업을 하나로 묶어서 '전부 성공시키거나', '하나라도 실패하면 전부 실패시키는것'을 의미한다.
