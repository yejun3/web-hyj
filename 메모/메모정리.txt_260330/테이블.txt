테이블
- **행과 열로 이루어진 데이터의 집합을 테이블이라고 한다.**
- 엑셀을 켜고 보는 모양과 매우 흡사하다.
- 일반적인 데이터베이스에서는 행과 열만 있으면 테이블이라고 하지만, **관계형데이터베이스에서는 특별한 제약을 추가해서 릴레이션(relation)이라고 부른다.**

***릴레이션(relation)은 테이블과 같은 말***  // 릴레이션 용어 기억할 것
 행(row) : 가로 데이터셋 , 튜플(tuple), 또는 레코드(record)라고 함  // 1명에 대한 정보
              행의 수를 기수(Cardinality) 라고 한다
열(column):  세로 데이터셋, 속성(attribute)이라고도 함 // 1가지에 대한 정보
               열의 수를 차수(Degree)라고 한다

도메인 (domain)
하나의 속성이 취할 수 있는 동일한 유형의 집합
속성에 들어갈  데이터값들의 동일한 유형의 집합
성별을 저장하기로 했으면 남녀에대한 데이터만 들어옴

자료형(데이터타입)

1.문자데이터 타입 
CHAR(N) 
//N은 저장할 수있는 문자의 길이

VARCHAR(N) 
// 가변길이타입
// CHAR(10) 이라면 어떤 길이의 문자열이 저장되더라도 그 값이 10만큼 저장 용량을 차지하지만 
// VARCHAR(10)의 경우 실제 저장된 값에 맞게 최적화 많이 사용됨

TEXT
- 문자열을 저장하는 타입으로 최대 65,535자까지 저장할 수 있다.

2.숫자 데이터 타입
정수형 타입
TINYINT
-128 ~ 127까지 저장

INT
- -21억 ~ +21억

실수형 타입
DECIMAL(M,D)
- 일반적으로 자주 쓰이는 실수형 타입 중 하나
- M : 최대로 쓸 수 있는 전체 숫자의 자리수
- D : 최대로 쓸 수 있는 소수점 자리수를 의미한다.
- ex) `DECIMAL(5,2)` 라면 -999.99 ~ 999.99를 나타낸다.

FLOAT
-3.40E+38 ~ -1.17E-38

DOUBLE
- FLOAT에 비해 더 넓은 범위의 수를 나타냄

3.날짜 테이터 타입
`DATE`
- 날짜를 저장하는 데이터 타입이다.
- 날자는 `2020-03-26` 이런 형식의 연,월,일 순으로 값을 나타낸다.

## 제약조건
- 제약조건이란, 테이블에 결함이 있는 데이터가 입력되지 않도록 미리 지정해둔 조건
제약조건은 테이블을 생성할 때 함께 설정
컬럼명 데이터타입 제약조건

- NOT NULL
컬럼명 데이터타입 NOT NULL
: 해당 컬럼에는 반드시 데이터를 입력해야 한다.

- UNIQE
컬럼명 데이터타입 UNIQUE; 
또는
 CONSTRAINTS 제약조건명 UNIQUE(컬럼명,...);
:중복되는 값을 허용하지 않는다.

- PRIMARY KEY(기본키)
- 보통 테이블에 키를 생성했다라고 할 때, 이는 기본키를 의미한다.
- 기본키는 UNIQUE와 NOT NULL 속성을 동시에 가진 제약조건으로, 테이블 당 1개의 기본키만 생성할 수 있다.

: 컬럼명 데이터타입 PRIMARY KEY;
또는
 CONSTRAINT 제약조건명 PRIMARY KEY(컬럼명,...);

- OREIGN KEY(외래키)
: CONSTRAINT 외래키명 FOREIGN KEY(컬럼명, ...)
: REFERENCES 참조 테이블(참조 테이블 컬럼명, ...);

// 부서의 정보는 부서테이블에 저장이 되어있고 사원 테이블에는 부서번호만 저장을 했을때 
    신입사원이 입사했을 때 잘못된 부서번호를 입력하는 오류를 범할 수 있다.
   외래키를 설정해놓으면 부서정보에 없는 부서번호를 입력하려고 할 때 오류를 발생시켜 잘못된           
   데이터가 입력되는 것을 방지한다.

외래키 제약사항
-외래키는 다른 테이블을 참조한다.  
반드시 참조하는 테이블이 먼저 생성되어야 하며, 
참조키가 참조 테이블의 기본키로 만들어져 있어야 한다.
- 외래키에 사용할 수 있는 컬럼 개수는 최대 32개이다.
- 여러 컬럼을 외래키로 만들려면, 참조하는 컬럼과 외래키 컬럼의 순서와 개수는 같아야 한다.

 CHECK
- 컬럼에 입력되는 데이터를 체크해 특정 조건에 맞는 데이터만 입력받고 그렇지 않으면 오류를 낸다.

CHECK 제약조건은 값이 들어올 때 검사
NULL이 들어오게 되면 비교자체가 안된다.
CHECK는 FALSE는 막는데 UNKNOWN은 통과

: CONSTRAINT 체크명 CHECK(체크조건);

DEFAULT
- 아무것도 입력하지 않으면 DEFAULT로 설정한 기본값이 입력됨
- 값이 입력되면 기본값은 무시된다.

: 컬럼명 데이터타입 DEFAULT 값
