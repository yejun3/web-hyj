Map인터페이스 
-key와 Value가 한 쌍으로 데이터로 저장된다는 것

HashMap : Map인터페이스를 구현하는 대표적인 클래스
Put(key,value):데이터삽입
ㄴ없으면 데이터 추가 , 있으면 value갱신이 일어남
get(key): 해당 key와 연결된 value 값을 반환

getOrDefault(key,default): 해당 key의 value가 있으면 반환,없으면 default값을  반환

containskey(key) : 해당 key가 포함되어있는지 검사
containsValue(value) : 해당 value가 포함되어있는지 검사

keySet(); :모든 map에 들어있는 모든 key만 추출하여 set에 저장
values(): map에 들어있는 모든 value들만 가져오기
entrySet(): map에 들어있는 데이터를 하나씩 꺼내서 set에 저장

향상된 for문, 람다식 -> 안써도 개발에는 지장 없음
ㄴ효율적인 코드작성에서 좋음

객체지향 프로그램에서의 메서드 함수 사용법
1.클래스 생성
2.크래스안에 메서드 작성
3.그 클래스의 객체를 생성
4.객체로부터 메서드를 호출

함수형 프로그래밍
함수를 정의하면 객체 생성없이 호출하는게 가능하다
다른 파일에 함수를 정의했다면 import만 하면 사용할 수 있다.

람다식(lambda expression) 장점
1. 간단하게 정리가능
2.핵심로직만 쓴다.

람다식을 구현하려면 함수형 인터페이스가 필요하다.
-추상 메서드가 1개만 있는 인터페이스

람다식 기본문법
-반환타입과 메서드명은 제거
(매개변수) -> {실행문}

매개변수가 1개일때 
-소괄호는 생략이 가능하다.
-매개변수의 타입 생략이 가능하다.

매개변수가 2개 이상일때 
-타입을 생략하려면 전부 다 생략해야 한다.
-소괄호 생략은 불가능하다.

반환할 값이 없고 명령이 1줄이면 중괄호 생략이 가능하다.
명령이 1줄이고 return과 중괄호를 같이 생략할 수 있다.

함수식을 변수에 담을 수있다.

변수를 통해서 함수를 호출

매개변수에 함수를 전달
반환값을 함수로 반환한다.
