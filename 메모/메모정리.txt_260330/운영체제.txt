총괄관리자 역할을 한다.

-목적에따라 어떤 운영채제의 종류와 버전을 선택할지 신중히 고민해야 함
-서버 개발에는 Linux, 가정용
  pc는 Windows를 사용

1-2 운영체제의 주요 기능

- 처리능력 향상
시간당  얼마나  많은 일을 처리했는가 

- 응답시간 단축
사용자가 요청한 작업에 대한 결과가 나오는 시간을 줄이는 것

- 신뢰도 향상
시스템 오류 없이 안정적인 동작 문제생겨도 복구하도록 하는 것

-인터페이스 제공
사용자와 컴퓨터가 소통할 수 있게 해주는 접점
- CLI(명령어 기반) → 터미널
- GUI(그래픽 기반) → Windows, macOS

CLI 환경도 자주 사용
- Git 명령어 사용
- Java 실행
- Node 서버 실행
- Linux 서버 접속
- `ls`, `cd` 같은 명령어
- 마우스로 폴더 클릭

- 입출력관리

- 프로그램 및 사용자 보호

관리자 권한 
- 운영체제 전체를 제어할 수있는 권한
-시스템 설정을 변경할수 있다.
윈도우 기준: 우클릭 -> 관리자권한으로 실행
특정 폴더에 접근
관리자 권한이 없어서 설치할 수없는 것들을 설치할 수 있다.

일반권한
-개인작업만 가능
-시스템에 영향을 주는 작업은 제한됨

주기억 장치 관리
일반적으로 RAM을 의미함

- 프로그램이 너무 많은 메모리를 사용하면 메모리 누수 발생
메모리누수 
사용이 끝난 메모리를 해제하지 못하고 계속 잡고있는 상태 
비유 : 컵에남아있는 물에 물을 또 따르고 넘침
- 서버가 점점 느려지고 실행 중 강제 종료 발생

개발자는 작업 관리자나 모니터링 도구를 통해 메모리 
사용량을 자주 확인하고 관리해야 한다.

C언어 프로그램 -> 개발자가 직접 메모리 할당솨 회수 가능
ㄴ어렵기 하지만 효율적으로 프로그램을 만들 수있다.

GC(가비지 컬렉터)
java -> JVM내부에 GC가 있음 -> 객체가 사용이 안되는거 같음 ->
메모리를 회수함

 CPU 관리
CPU는 컴퓨터에서 실제 연산과 처리를 수행하는 핵심 장치
운영체제는 CPU를 어떤 프로그램에 먼저 사용할지 결정한다. 
(이 과정을 CPU 스케줄링이라고 한다.)

CPU는 한번에 하나의 작업만 실행한다.
-엄청빠르게 번갈아 실행하기 때문에 동시에 실행하는 것처럼 보임

코어
-CPU 안에 있는 실제 계산하는 장치
-하나의 코어 -> 하느이 "작업처리 유닛"

스레드
-프로그램 안에서 실행되는 작업흐름   

 파일 관리
- 파일 관리는 **파일을 만들고, 저장하고, 삭제하고, 찾고, 보호하는 기능**

로그 (log)
프로그램이 실행되면서 남는 기록
-문제해결의 가장 중요한 단서

 명령어 해석 및 실행

- 사용자가 입력한 명령어를 해석하고 실행하는 기능
- `mkdir test` → 폴더 생성
- `rm file.txt` → 파일 삭제

운영체제 주요기능
1.처리능력의 향상
-주어진 시간 내 얼마나 많은 일을 처리하는가
2.응답시간의 단축
-사용자가 요청한 작업에 대해 결과가 나올때까지 걸리는시간 단축
3.신뢰도 향상 
-시스템이 오류없이 안정적으로 동작하고 문제가 생겨도 잘 복구 되도록 하는 것
4.인터페이스 제공
-사용자와 컴퓨터가 소통할 수 있게 해주는 접점
5.입출력 관리 
-입력 장치와 출력장치가 정상적으로 동작하도록 관리한다.
6.프로그램 및 사용자 보호
-프로그램끼리 서로 함부로 침범하지 못하게 하고, 사용자 데이터도 안전하게 지키는 기능
7.주기억장치(RAM)관리 
-필요한 프로그램에 메모리를 할당하고 사용이 끝나면 회수한다.
8.CPU 관리 
운영체제가 CPU를 어떤 프로그램에 먼저 사용할 지 결정
9.파일관리
-파일을 만들고, 저장하고, 삭제하고 찾고, 보호하는 기능
10.명령어 해석 및 실행
-사용자가 입력한 명령어를 해석하고 실행하는 기능이다.

1-3. 운영체제의 종류

- 운영체제는 사용하는 목적과 환경에 따라 다르게 선택된다.
- 개인용 컴퓨터 : 윈도우
- 기업용 서버 : 유닉스, 리눅스
- 스마트폰 : iOS, 안드로이드

- *어떤 기기에서*, *어떤 목적으로*, *어떤 방식으로 사용되는지*에 따라 종류가 나뉜다.

운영체제를 선택하는 것이 중요한 이유
1) 개발 환경이 달라진다.
2) 배포 환경이 달라진다.
3) 개발 대상이 달라진다.

윈도우(Windows)
가장 큰 특징은 *그래픽 사용자 인터페이스(GUI)* 가 잘 갖추어져 있는 점
마우스로 클릭하고 창을 열고 닫는 방식으로 쉽게 사용할 수 있다.

유닉스(UNIX)
- 대용량 처리와 높은 안정성이 필요한 서버 환경에서 사용되는 운영체제

특징*
- 안정성이 높습니다
    - 서버는 24시간 동작해야 하므로 안정성이 매우 중요합니다.
- 보안성이 좋습니다
    - 기업 시스템은 데이터 보호가 중요하므로 보안성이 필요합니다.
- 대규모 처리에 적합합니다
    - 많은 사용자와 많은 데이터를 처리하는 환경에서 사용됩니다

 리눅스(Linux)
- **리눅스는 오픈소스 운영체제이다.**

오픈소스
-소스코드가 공개되어 있는 소프트웨어
-누구나 자유롭게 열람 , 수정 , 배포할 수있다.

오픈소스 사용하면 좋은 이유 
1. 비용절감
-대부분 무료, 기업에서도 적극적으로 활용
2. 빠른 개발
-로그인 기능 직접 구현하기보다 라이브러리를 쓰는게 좋다.
->개발속도가 폭발적으로 증가
3.검증된 코드
-많은 개발자가 사용
-버그 빠르게 발견
-안정성이 높다.

4.커뮤니티 지원
-검색하면 웬만하면 해결방법이 있음
-문서 예제 풍부하다.

라이브러리
-클래스나 인터페이스 묶음
ㄴ필드,매서드를 정의해놓을 수있다.
-다른 프로젝트에서 라이브러리를 import하여 필드나 메서드를 사용할 수 있다.
임베디드
-특정 기능을 수행하기 위해 기기 안에 내장된 컴푸터 시스템
전자레인지,세탁기,자동차,티비,에어컨 등등
