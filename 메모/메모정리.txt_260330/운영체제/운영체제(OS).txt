- 하드웨어와 소프트웨어 중간을 담당하는 프로그램
- Windows, MacOS, iOS,Android, Linux, Unix

- 개발하려는 목적에 따라 운영체제를 선택을 잘 해야 한다.

- 앱 개발 : MacOS(애플 전용 앱),

운영체제가 하는일
- 주어진 시간 내에 많은 일을 처리하기
- 사용자 요청에 최대한 빨리 응답하기
- 오류 없이 안정적으로 동작하게 하기
- 사용자에게 인터페이스 제공하기
- 입출력장치 충돌나지 않게 관리
- 프로그램 및 사용자 보호하기
- 주 기억장치 관리
- cpu관리
- 파일관리(생성,삭제,수정)
- 명령어 실행


한 컴퓨터에서 인터넷 브라우저, 문서 편집기, 메신저가 동시에 실행되고 있다.

파일 소유권
- 어떤 사용자와 어떤 그룹이 그 파일의 주인인가를 나타내는 정보

리눅스에서 파일 하나에는 2가지 소유 정보가 붙는다.

소유 사용자(User Owner)
- 해당 파일의 대표 사용자

소유 그룹(Group Owner)
- 해당 파일과 관련된 대표 그룹
- 같은 그룹에 속한 사용자들은 그룹 권한 범위 안에서 접근할 수 있다.

왜 파일 소유권이 필요한가
- 보안과 역할 분리 때문이다.

파일 허가권
"누가 이 파일에 대해, 무엇을 할 수 있는가"를 정하는 규칙

컴퓨터를 여러 사용자가 함께 쓰거나, 서버에서 여러 프로그램이 동시에 동작하면 아무나 파일을 읽고, 수정하고, 실행할 수 있으면 큰 문제가 생긴다.

누가 접근하는가
user : 파일의 소유자
group : 파일이 속한 그룹
other : 그 외 모든 사용자

무엇을 할 수 있는가
권한의 종류는 3가지 이다.
r : read, 읽기
w : write, 쓰기
x : execute, 실행

어디에 적용이 되는가
- 일반 파일
- 디렉터리(폴더)

파일에서의 r,w,x의 의미
r(read)
- 파일의 내용을 읽을 수 있다.
- 텍스트 파일 열기
- cat을 이용하여 내용 보기
- 프로그램이 파일 내용 읽게하기

w(write)
파일의 내용을 수정할 수 있다.
- 내용 추가
- 내용 삭제
- 내용 덮어쓰기

x(execute)
- 파일을 실행할 수 있다.
- 쉘 스크립트 실행
- 실행파일 실행

디렉터리에서 r,w,x의 의미
r : 디렉터리 안에 어떤 파일이 있는지 목록을 볼 수 있다.
w : 디렉터리 안에서 파일 생성, 삭제, 이름 변경
x : 디렉터리 안으로 들어갈 수 있는 권한

- rw- r-- r--
첫글자 : 파일의 종류
- : 일반 파일
d : 디렉터리
l : 링크

rw- : 소유자는 읽기, 쓰기만 가능하다.
r-- : 그룹은 읽기만 가능
r-- : 다른 사용자는 읽기만 가능

권한을 숫자로도 표현할 수 있다.
우분투에서는 권한을 문자 대신 숫자로 표현하기도 한다.

각 권한에 숫자를 부여한다.
r = 4
w = 2
x = 1

이 값을 더해서 표현한다.
rwx = 4 + 2 + 1 = 7
rw- = 4 + 2 = 6
r-- = 4
r-x = 4 + 1 = 5

허가 변경하기

chmod 644 memo.txt
소유자 : rw-
그룹 : r--
기타 : r--

문자 방식
u : user
g : group
o : other
a : all
연산자
+ : 권한 추가
- : 권한 제거
= : 권한을 지정한 값으로 설정

chmod u+x test.txt
소유자에게 실행 권한을 추가

chmod g-w memo.txt
그룹의 쓰기 권한을 제거

chmod o-r memo.txt
기타 사용자의 읽기 권한을 제거
