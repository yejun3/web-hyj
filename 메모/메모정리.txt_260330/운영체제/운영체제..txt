1-2  운영체제 주요 기능

1. 처리 능력의 향상
- 주어진 시간 내 얼마나 많은 일을 처리하는지

2. 응답시간의 단축
- 사용자가 요청한 작업에 대해 결과가 나올때까지 걸리는 시간

3. 신뢰도 향상
- 시스템이 오류없이 안정적으로 동작하고, 문제가 생겨도 잘 복구되도록 하는 것

4. 인터페이스 제공
- 사용자와 컴퓨터가 소통할 수 있게 해주는 접점
- CLI(명령어 기반) → 터미널
- GUI(그래픽 기반) → Windows, macOS


5. 입출력 관리
- 입력 장치와 출력 장치가 정상적으로 동작하도록 관리한다.

6. 프로그램 및 사용자 보호
- 프로그램끼리 서로 함부로 침범하지 못하게 하고, 사용자 데이터도 안전하게 지키는 기능이다.

7. 주기억 장치(RAM) 관리
- 필요한 프로그램에 메모리를 할당하고, 사용이 끝나면 회수한다.

8. CPU 관리
- 운영체제가 CPU를 어떤 프로그램에 먼저 사용할지 결정

9. 파일관리
- 파일을 만들고, 저장하고, 삭제하고 찾고, 보호하는 기능

10. 명령어 해석 및 실행
- 사용자가 입력한 명령어를 해석하고 실행하는 기능이다.

1-3. 운영체제의 종류

- 운영체제는 사용하는 목적과 환경에 따라 다르게 선택된다.
- 개인용 컴퓨터 : 윈도우
- 기업용 서버 : 유닉스, 리눅스
- 스마트폰 : iOS, 안드로이드

- *어떤 기기에서*, *어떤 목적으로*, *어떤 방식으로 사용되는지*에 따라 종류가 나뉜다.

운영체제를 선택하는 것이 중요한 이유
1) 개발 환경이 달라진다.
2) 배포 환경이 달라진다.
3) 개발 대상이 달라진다.

wsl(Windows SubSystem for Linux)
윈도우 안에서 리눅스 환경을 실행할 수 있게 해주는 기능
윈도우를 그대로 쓰면서도 bash, ls, pwd, apt, git과 같은 리눅스 명령어와 도구를 사용할 수 있다.
전통적인 가상머신이나 듀얼부팅 없이 Windows에서 GNU/Linux 환경을 직접 실행할 수 있다.

가상머신보다 가볍고 편하기 때문에 사용을 한다.

wsl을 통해 리눅스를 설치하게 되면 Linux 커널을 기반으로 만들어진 Ubuntu가 설치가 된다.

sudo(Superuser Do)
- 리눅스에서 관리자 권한으로 실행해라 라는 뜻이다.

우분투에서는 일반 사용자가 아무 프로그램이나 시스템에 설치하거나 바꾸지 못하게 막아둔다.
- 실수로 시스템이 망가지는 것을 방지
- 보안 강화
- 중요한 파일 보호

시스템 전체에 영향을 주는 명령어는 보통 sudo가 필요하다.

apt
우분투에서 프로그램을 관리하는 도구이다.
정확히는 패키지 관리자 라고 부른다.
- 프로그램
- 라이브러리
- 설치파일의 묶음
- 업데이트 파일

윈도우에서는 인터넷에 접속하여 .exe파일을 직접 다운로드해서 설치한다.

우분투는 apt를 통해 설치를 한다.

update : 패키지 목록 정보를 최신으로 갱신하는것

sudo apt update : apt가 가서 다운받는 저장소의 목록을 갱신한다.

설치 목록이 오래됐을 수 있기 대문이다.
설치 오류가 날수도 있다.
구버전을 설치하게 될 수도 있다.

저장소에서 정보를 받아와야 하므로 인터넷이 연결되어 있어야 한다.

시스템 패키지 정보에 영향을 주기 때문에 보통 sudo를 붙여야 한다.
