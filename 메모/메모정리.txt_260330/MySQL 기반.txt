스키마 = 데이터베이스
데이터를 어떻게 저장할 것인지에 대한 설계도

스키마의 구성요소
-테이블(table)
-뷰(view)
-인덱스(index) 
-트리거(trigger)
-저장 프로시저(Procedure)

스키마를 사용하는 이유

1.데이터 구조를 정리하기 위해
-구조를 정해서 일관성을 유지

2. 데이터 관리 단위로 사용하기 위해
-프로젝트 별로 DB를 나눌 수 있다.
-예: 쇼핑몰/ 게시판 / 회원관리

3.접근 권한 관리
-특정 스키마만 접근 하도록 설정 가능

데이터베이스는 좀 더 깊은 개념으로 3가지로 나눈다.

1.외부 스키마
-사용자 입장에서 보는 구조
EX) 화면에 보이는 데이터

2.개념 스키마
-전체 데이터 구조
-DB설계자 관점

3.내부 스키마
-실제 저장방식
-디스크에 어떻게 저장되는지

Data : 가공되지 않은 단순한 사실이나 값
DB : 데이터를 체계적으로 저장해놓은 공간
DBMS : 데이터베이스를 관리하는 프로그램(소프트웨어)

DBMS의 종류
1.계층형 데이터베이스
- 트리 구조로 데이터를 저장하는 방식
- 1:N 관계만 가능하다.
- 구조가 단순하다.
- 변경이 어렵다.

2.네트워크형 데이터베이스
- 데이터를 그래프 형식으로 연결하는 방식
- N:N 관계가 가능하다.
-계층형보다 유연하다.
-구조가 복잡하다.
-여러개의 연결을 허용하는 DB

3.관계형 데이터베이스
-테이블 형태로 데어터를 저장하는 방식
-가장 많이 사용됨
-SQL을 사용
-데이터의 무결성을 보장한다.

4.객체형
-객체(Object) 형태로 데이터를 저장
-Java, C++ 객체 그대로 저장 가능하다.
-복잡한 데이터 처리에 유리
-사용 빈도는 낮다.

5.NoSQL(Not only SQL)
-관계형이 아닌 다양한 방식의 DB

key - value

Document
-JSON

Column
-컬럼 중심 저장

Graph
-관계중심

-대용량 처리에 강함
-속도가 빠름
-유연한 구조

SQL(Structured Query Language)
-DBMS 상에서 데이터를 읽고 쓰고 삭제하는등 데이터를 관리하기위한
 일종의 프로그래밍 언어이다.

DDL(Data Definition Language) : 데이터 정의어
-데이터 베이스 객체를 관리하는 언어
-테이블, 속성, 제약조건, 뷰
-CREATE : 데이터베이스 객체 사용에 사용
-DROP : 삭제에 사용
-ALTER : 수정에 사용
-TRUNCATE : 테이블은 유지하고 데이터를 통째로 삭제

데이터베이스 생성
CREATE DATABASE IF NOT EXISTS 데이터베이스 이름
CHARATER SET = utf8mb4
COLLECT = utf8mb4_unicode_ci

CHARATER SET 
이 데이터베이스에서 문자를 저장할 기본 문자셋(Endcoding)을 지정
utf8mb4가 MySQL에서 가장 안전한 유니코드 문자셋이다.

COLLATE 
정렬과 비교의 기준을 설정

테이블
-행과 열로 이루어진 데이터의 집합
-테이블끼리 관계를 맺고, 제약조건이 추가되서 릴레이션이라고도 불린다.

행(row)
-가로로 묶은 데이터 셋
-튜플, 레코드라고도 부른다.
-행의 개수를 '카디널리티'라고 부른다.

열(column)
-세로로 묶은 데이터 셋
-그 테이블의 속성(attribute)을 의미한다.

자료형
문자형
CHAR : 고정형 
VARCHAR : 가변형 길이
TEXT : 대용량 문자열

숫자형
INT
FLOAT
DOUBLE

날짜형
DATE
DATETIME
TIMESTAMP

제약조건
-문제가 되는 데이터를 테이블에 추가할 수 없도록 제한하는 조건
-테이블을 만들면서 수정할 수 있고, 추후에 생성하거나 변경할수도 있다.

PRIMARY KEY (기본키)
-데이터의 값이 있어야 하고, 겹쳐서는 안된다.

Not Null 
-해당 속성에는 반드시 값이 들어가야한다.

UNIQUE
-속성에 들어가는 값은 유일해야 한다.

FOREIGN KEY (외래키)
-다른 테이블의 기본키 값을 참조하는 제약조건

CHECK
-속성에 들어갈 수 있는 데이터의 조건을 설정하는 제약조건

DEFAULT
-값을 입력하지 않았을 때 자동으로 들어가는 기본값을 설정

CREATE TABLE 테이블명( 
          속성명 자료형,
          속성명 자료형,
          속성명 자료형 //마지막은 ,를 붙이지 말것
);

데이터베이스 객체의 수정
ALTER 

컬럼의 추가
ALTER TABLE 테이블명 ADD COLUMN 컬럼명 자료형;

컬럼의 삭제 
ALTER TABLE 테이블명 DROP COLUMN 컬럼명;

컬럼명 변경
ALTER TABLE 테이블명 RENAME COLUMN 컬럼명 TO 변경할 이름;

자료형의 변경
ALTER TABLE 테이블명 MODIFY COLUMN 컬럼명 자료형;

제약조건 추가
ALTER TABLE 테이블명 ADD 제약조건 (컬럼);

제약조건 삭제
ALTER TABLE 테이블명 DROP 제약조건 종류(컬럼);
-기본키의 경우는 이름없이 삭제가 가능
-나며지 제약조건은 이름을 명시해줘야 한다.

-NOT NULL과 DEFAULT는 MODIFY를 사용한다.
ALTER TABLE 테이블명 MODIFY 컬럼명 데이터타입;

무결성 
-데이터베이스에 저장된 데이터가 현실 세계의 실제값과 일치한느 정확성을 의미한다.

1.개체 무결성
-PK로 설정된 컬럼은 반드시 고유한 값을 가지고 있어야 한다. 그리고 NULL이 될 수 없다.

2.참조 무결성
-외래키(FK)값은 NULL이거나, 반드시 참조하는 테이블의PK에 들어있는 값과 일치해야 한다.
-존재하지 않는 값은 외래키로 저장할 수 없다.

3.도메인 무결성 
-특정 속성의 값이 미리 정의된 유효한 값 범위 안에 있어야 한다.

4.사용자 정의 무결성
-사용자가 논리적으로 필요한 규칙을 직접 정의하는것
-나이는 음수가 들어올 수 없다. 가입일은 오늘 이후일수 없다.
