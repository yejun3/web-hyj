컬렉션 프레임 워크
프레임워크 : 인터페이스나 클래스를 준비해놓고 사용한는것 

인터페이스
-상수
-추상메서드

List 인터페이스
- 순서가 있다.(인덱스가 존재한다.)
-중복이 가능하다.
-넣는대로 들어간다.
ArrayList : List 인터페이스를 구현한 대표적인 클래스
ㄴ조회에는 유리하다. 추가 삭제에는 불리하다.
LinkedList...

Set
-순서가 없다.(인덱스가 없기 때문에)
- 중복이 안됨(중복된 내용을 추가하려해도 반영이 안됨)
- HashSet :Set 인터페이스를 구현한 대표적인 클래스

반복자(Iterator)
Set같은 index가 없는 컬렉션에서 데이터를 하나씩 꺼내주는 방법
반복자 객체는 컬렉션 객체로부터 iterator()메서드를 사용해 생성이 가능하다
List<Integer>list = new ArrayList (컬렉션객체 생성)
list.iterator (매서드 호출)

hasNext(): 다음에 반환할 요소가 있으면 true/아니면 false
ㄴ결과가 true/아니면 false로 나온다는 것은 조건식(if), 반복문의 조건에 들어갈수있다.
next(): 컬렉션에서 요소를 하나 반환

향상된 for문도 내부에서 반복자를 기반으로 동작을 한다.

Collections 클래스
-컬렉션을 사용하는데 있어 유용한 기능들을 가지고 있는 클래스
-직접 객체를 생성하지 않는다.
-모든 메서드는 static 선언이 되어있다.
-데이터를 저장하기 보다는 가공/처리한는 역할

정렬 : sort()
뒤집기:reverse()
섞기: sufle()
최댓값: Max()
최소값: Min()
