OSI 7 계층
컴퓨터끼리 통신할 떄 어떤 일을 단계별로 나눠서 처리하는 구조
데이터를 한 번에 막 보내는게 아니라 역할을 7단계로 나눠서 보내고 받도록 정리한 개념

OSI 7계층이 필요한 이유
예전에는 컴퓨터 제조사나 시스템마다 통신방식이 제각각이었다.
서로 통신이 어려웠다.
국제 표준화기구 (ISO)가 통신 과정을 단계별로 나눠서 표준처럼 이해하자

1.통신과정을 쉽게 이해할 수 있다.
-복잡한 통신을 한 번에 보면 어렵다.
-기능을 층별로 나눠서 본다.
2.문제가 생겼을 때 어디서 고장이 났는지 찾기 위해서
-예를 들어 인터넷이 안 될 때
-케이블 문제인지
-IP문제인지
-프로그램 문제인지 
구별해서 볼 수 있게 된다.

3. 장비와 기술의 역할을 정리하기 위해 
- 허브, 스위치 , 라우터 ,IP ,TCP, 브라우저, 웹서버와 같은 것들이 어느 계층에서 동작하는지 구분할 수 있다.

각 계층
1계층 : 물리 계층 (Physical Layer)
-데이터를 실제로 보내는 계층
-전기 신호, 빛, 무선 신호처럼 눈에 보이지 않는 실제 신호로 데이터를 전달한다.
-랜선, 광케이블, 허브, 리피터, 무선전파, 커넥터
허브(여러 컴퓨터를 연결하는 장치)
리피터: 약해진 네트워크 신호를 다시 강하게 만들어주는 장치

이 계층의 특징
-데이터의 의미는 모른다.
-그냥 신호를 보낸다.
-문자인지 이미지인지는 식별 불가능
-오직 비트(0,1) 수준으로 처리
  
택배를 실은 트럭과 유사
안에 뭐가 있는지 모르지만 택배가 있는 거는 암

2계층: 데이터 링크 계층(Data Link Layer)
같은 네트워크 안에서 데이터를 제대로 가도록 처리하는 계층
MAC 주소를 사용한다.

MAC 주소란?
-네트워크 카드에 부여된 고유한 물리 주소이다.
-장비 자체의 주소라고  보면된다.

내 PC의 랜카드 주소 
스위치가 장비를 구분할 때 사용하는 주소

이 계층의 주요 역할 
-프레임 단위로 데이터 구성
-같은 Lan  안에서 목적지 찾기
-오류 확인 
-MAC 주소 기반 전달

대표장비 
스위치: 여러 장치를 연결하면서 필요한 장치에게만 데이터를 전달하는 네트워크 장비
-목적지 장치에만 전송
-충돌 거의 없음
- 속도 빠름

허브는 모든 장치에게 데이터를 전송(비효율)
충돌발생많다.
보안 취약

택배 상자에 "이 건물(스위치)안에서 누구에게 전달할지"를 적는 것

3계층 : 네트워크 계층(Network Layer)
-서로 다른 네트워크를 넘어 목적지까지 가는 길을 찾는 계층
여기서 가장 중요한 것이 IP주소이다.

IP주소
인터넷이나 네트워크에서 장치를 식별하기 위한 논리적 주소이다.

이 계층의 핷미 역할
-목적지  IP주소 확인
-어느 경로로 갈지 결정
-여러 네트워크를 거쳐 전달

대표장비
라우터(Router)
- 서로 다른 네트워크를 연결하고 데이터를 목적지까지 가장 적절한 경로로 보내는 장치

2계층이 "같은 건물 안에서 누구에게 줄까"라면 
3계층은 "서울에서 부산까지 어떤 길로 갈까"를 결정하는 단계이다.

LAN(Local Area Network)
좁은 범위의 네트워크
-집 
-학교
-회사 내부
WAN(Wide Area Network)
넓은 범위의 네트워크
-국가
-전 세계

MAN(Metroplitan Area Network)
넓은 범위의 네트워크
- 국가
-전 세계

4계층 : 전송계층(Transport Layer)
-데이터를 정확하고 안정적으로 전달할지 , 아니면 빠르게 보낼지 결정하는 계층

TCP
UDP

이 계층의 핵심역할
-데이터 분할
-순서 맞추기
-누락확인
-재전송
-흐름제어
-포트번호 사용

포트 
한 컴퓨터 안에서도 여러 프로그램이 동시에 통신한다,
그래서 어느 프로그램으로 보내야 하는지 구분하는 번호

80번포트 : HTTP
443포트 :  HTTPS

5계층 : 세션 계층(Session Layer)
-통신 연결을 시작하고, 유지하고, 끝내는 역할을 한다.

세션 
통신 흐름의 연결 상태

역할
-연결 시작
- 연결 유지
- 연결 종료
- 대화 동기화

세션은 마치 전화 통화 같은 것
전화를 건다 - 연결시작 
대화를 한다 - 세션의 유지
전하를 끊는다 - 세션을 끊는다.

왜 필요한가 
- 두 프로그램이 데이터를 주고받으 떄 아예 연결상태를 관리하지 않으면 중간에 끊겨는지, 계속 대화중인지 파악하기 어렵다.

6계층 : 표현 계층(Presentation Layer)
-데이터를 서로 이해할 수 있는 형태로 바꾸는 계층

보내는쪽과 받는 쪽의 데이터 표현방식이 다를 수 있기 떄문에 그 차이를 맞춰주는 역할을 한다.

주요역할
-데이터 역할 변환
-문자 인코딩 처리
-암호화/복호화
-압축/해제

7계층: 응용 계층(Application Layer)
사용자가 실제로 접하는 네트워크 서비스 계층이다.
우리가 프로그램을 통해 사용한는 웹,이메일,파일전송과 같은 서비스가 이 계층과 가깝다.

보낼 때 
ex) 사용자가 웹 브라우저에서 사이트 접속 요청을 한다.
1.응용계층 
-브라우저가 HTTP 요청을 만든다.
2.표현계층
-필요한 형식 변환, 암호화 등을 처리한다.
3.세션 계층
-통신 연결 흐름을 관리한다.
4.전송계층
-TCP를 사용해 데이터를 잘게 나누고 포트 번호를 붙인다.
5.네트워크 계층
-IP주소를 붙여 목적지를 정한다.
6. 데이터 링크 계층
-Ma           
c 주소를 붙여 같은 구간에 전달할 수 있게 한다.
7.물리 계층
-전기 신호나 무선신호로 실제 전송한다.

받는 쪽 
1. 물리계층에서 신호를 받는다.
2.데이터 링크 계층에서 플레임을 확인한다.
3.네트워크 계층에서 IP를 확인한다.
4.전송계층에서 TCP/UDP 포트를 확인한다.
5.세션 계층에서 연결 상태를 본다.
6.표현 계층에서 해석 가능한 형식으로 바꾼다.
7.응용 계층에서 최종적으로 프로그램이 데이터를 사용한다.
