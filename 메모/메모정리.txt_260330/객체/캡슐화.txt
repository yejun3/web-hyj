캡슐화
데이터 와 기능을 하나로 묶고 
외부에서내부의 데이터를 직접 접근하지 못하게 제한하는 것을 말한다


다형성 
여러 클래스에서 하나의 메서드가 다른 방식으로 구현되는것을 말한다.
속성이나 기능이 맥락에 따라 여러 다른 역할을 수행할 수있다.
하나의 타입으로 여러 객체를 다룰 수 있는 성질
타이어 마다 성능이 다르다.
오버로딩과 오버라이딩이 다형성의 예시 

상속 
부모클래스의 필드와 메서드를 자식클래스가 상속받아 쓸수 있다.
코드의 중복을 줄여주고 한번의 수정으로 자식클래스 전체에게 반영되어 수정이 용이하다
extends키워드를 사용한다.
자식 클래스 뒤에 extends를 쓰고 부모클래스를 쓰면 상속이 된다.
이때 부모클래스가 2개 이상이 될 수는 없다
하지만 c가 a를 상속받은 b 클래스를 상속받는 것은 가능하다.

추상화
공통적인 속성과 기능을 추출한 것으로 단순화 시킨것
추상화의 구현에는 추상클래스와 인터페이스등이 있다.


오버로딩 
같은 매서드에서 매개변수를 다르게 정의한 것을 말한다.
컴파일 단계에서 실행 매서드명은 같아야하지만  매개변수의 갯수나 타입이나 순서가 달라야 한다.

오버라이딩 
부모클래스의 매서드를 재정의하는것
런타임에서 실행된다. 매서드명과 반환형 매개변수는 같아야한다

메서드
특정 기능을 수행하는 함수
메서드의 코드 중복을 없애고 츠로그램을 구조화한다.
접근제한자 반환형 메서드명(매개변수){
//실행할 코드
return 반환값;
}
인 형태이다.

구성요소
접근제한자: 외부에서 접근허용가능 범위를 제한한다.
반환형 : 반환값의 타입과 동일해야 하며 반환값이 없을 때는 void를 쓴다.
매개변수: 매서드 내에서 쓰일 변수를 지정한다. 이때 매개변수는 없을 수도 있다.
{}: 매서드가 실행할 부분을 나타내는 코드 블록이다.
return 반환값; : 메서드가 돌려줄 값을 나타낸다. 반환할 내용이 없으면 적지 않아도 된다.
