데이터 베이스
스키마(데이터베이스)
MySQL에서 '스키마'와 '데이터베이스'가 거의 같은 의미로 사용

특징
- 데이터 구조 정의 : 컬럼에 들어갈 데이터의 형식과 제약조건 정의
- 설계도 역할, 사용자 관리를 함

종류
외부스키마 : 사용자가 데이터베이스를 어떻게 인식하는지에 대한 view를 정의
개념스키마 : 데이터 베이스의 전체구조를 정의 , 데이터 간의 관계와 보안을 포함
내부스키마 : 데이터가 물리적 저장 장치에 어떻게 저장되고 있는가를 정의 (데이터타입, 저장위치)

SQL문
데이터 관리를 위한 프로그램 언어
대소문자를 구별하지 않는다.
세미콜론으로 문장 끝을 맺어야 한다.

SQL문장 종류
1.DDL : 데이터 정의어 // 객체관리
2.DML : 데이터 조작어 // 삽입,조회,삭제,갱신
3.DCL : 데이터 제의어 // grant, revolk 키워드
4.TCL : 트랜잭션 제의어 // commit, rollback
